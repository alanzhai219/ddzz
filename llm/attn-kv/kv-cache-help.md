# Single Head Attention: 无KV Cache vs 有KV Cache 对比分析
## Decode阶段计算量节省详解

---

## 📋 核心结论速览

| 步骤 | 无KV Cache | 有KV Cache | 节省效果 |
|------|-----------|-----------|---------|
| **Q投影计算** | ✅ 每步计算 | ✅ 每步计算 | ❌ 无节省 |
| **历史K投影** | ✅ 每步重新计算 | ❌ 从Cache读取 | ✅ **大幅节省** |
| **历史V投影** | ✅ 每步重新计算 | ❌ 从Cache读取 | ✅ **大幅节省** |
| **Q×Kᵀ注意力分数** | 计算全部N×N | 计算1×(N-1) | ✅ **O(N²)→O(N)** |
| **Attention×V聚合** | 计算全部序列 | 仅聚合历史缓存 | ✅ **大幅节省** |

---

## 🔍 详细计算步骤分解

### Single Head Attention 标准公式
$$\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

其中输入序列长度为 $N$，隐藏维度为 $d$。

---

### 📊 无KV Cache的Decode阶段（第 $t$ 步）

```
输入: 当前要生成的第 t 个token的embedding x_t ∈ ℝ^d
当前序列长度: t (包含prompt + 已生成的t-1个token)

步骤1: Q/K/V投影 (3次线性变换)
  - Q = x_t × W_Q          → O(d²)
  - K = [x_1...x_t] × W_K  → O(t·d²)  ← 重复计算历史!
  - V = [x_1...x_t] × W_V  → O(t·d²)  ← 重复计算历史!

步骤2: 注意力分数计算
  - S = Q × Kᵀ             → O(t·d)   (1×t 矩阵乘法)
  
步骤3: Softmax归一化
  - A = softmax(S/√d_k)    → O(t)

步骤4: 加权聚合
  - Output = A × V         → O(t·d)

总计算量: O(t·d² + t·d) ≈ O(t·d²)
```

**关键问题**：每一步都要重新计算**所有历史token**的K/V投影，导致：
- 生成 $T$ 个新token的总复杂度：$\sum_{t=1}^{T} O(t·d²) = O(T²·d²)$ [[30]]

---

### ✅ 有KV Cache的Decode阶段（第 $t$ 步）

```
输入: 当前token的embedding x_t ∈ ℝ^d
Cache: 已存储 [K_1...K_{t-1}], [V_1...V_{t-1}]

步骤1: 仅计算当前token的Q/K/V
  - Q = x_t × W_Q          → O(d²)
  - k_t = x_t × W_K        → O(d²)   ← 仅当前!
  - v_t = x_t × W_V        → O(d²)   ← 仅当前!
  
步骤2: 更新Cache (追加操作)
  - K_cache = concat(K_cache, k_t)   → O(1) 指针操作
  - V_cache = concat(V_cache, v_t)   → O(1) 指针操作

步骤3: 注意力分数计算 (仅当前×历史)
  - S = Q × K_cacheᵀ       → O((t-1)·d)  ← 1×(t-1) 矩阵乘法
  
步骤4: Softmax归一化
  - A = softmax(S/√d_k)    → O(t)

步骤5: 加权聚合
  - Output = A × V_cache   → O((t-1)·d)

总计算量: O(d² + t·d) ≈ O(d² + t·d)
```

**核心节省**：
1. **避免冗余K/V投影**：历史token的 $K, V$ 只需计算一次，后续直接从Cache读取 [[1]][[3]]
2. **注意力计算降维**：$Q×K^T$ 从 $t×t$ 降为 $1×(t-1)$，复杂度从 $O(t²·d)$ 降至 $O(t·d)$ [[10]]

生成 $T$ 个新token的总复杂度：$O(T·d² + T²·d)$，当 $d \gg T$ 时接近线性 [[30]]

---

## 🎯 具体节省的计算量量化

假设：`d=4096`, 生成 `T=100` 个token

| 操作类型 | 无Cache总计算量 | 有Cache总计算量 | 节省倍数 |
|---------|---------------|---------------|---------|
| **线性投影** (K/V) | $2×d²×\sum_{t=1}^{100}t ≈ 41M$ | $2×d²×100 ≈ 3.3M$ | **~12×** |
| **Q×Kᵀ矩阵乘** | $d×\sum_{t=1}^{100}t² ≈ 1.4B$ | $d×\sum_{t=1}^{100}(t-1) ≈ 20M$ | **~70×** |
| **Attention×V** | $d×\sum_{t=1}^{100}t² ≈ 1.4B$ | $d×\sum_{t=1}^{100}(t-1) ≈ 20M$ | **~70×** |

> 💡 实际推理中，KV Cache可使decode阶段速度提升 **5-10倍** [[9]]

---

## ⚠️ KV Cache的代价

虽然计算量大幅减少，但需要付出：

1. **内存开销**：
   ```
   Cache大小 = 2 × 层数 × head数 × head_dim × 序列长度 × 精度字节
   示例: Llama-7B, seq=4096, FP16 → ~2GB/序列 [[3]]
   ```

2. **内存带宽压力**：
   - Decode阶段变为**memory-bound**而非compute-bound [[3]]
   - 每步需读取整个KV Cache参与注意力计算

3. **批处理复杂度**：
   - 不同序列长度导致Cache管理复杂（需padding或PagedAttention）[[3]]

## 🔄 流程对比图

```
无KV Cache (第3步解码):
┌─────────────────────────┐
│ 输入: [x₁,x₂,x₃]         │
│ 1. 计算 Q₁,Q₂,Q₃        │ ← 重复!
│ 2. 计算 K₁,K₂,K₃        │ ← 重复!  
│ 3. 计算 V₁,V₂,V₃        │ ← 重复!
│ 4. Q₃ × [K₁,K₂,K₃]ᵀ    │
│ 5. Attention × [V₁,V₂,V₃]│
└─────────────────────────┘

有KV Cache (第3步解码):
┌─────────────────────────┐
│ Cache: [K₁,K₂], [V₁,V₂] │ ← 前两步已存
│ 输入: x₃                 │
│ 1. 仅计算 Q₃, k₃, v₃    │ ← 仅当前!
│ 2. Cache追加: +[k₃],[v₃]│
│ 3. Q₃ × [K₁,K₂,k₃]ᵀ    │ ← 复用历史
│ 4. Attention × [V₁,V₂,v₃]│
└─────────────────────────┘
```
